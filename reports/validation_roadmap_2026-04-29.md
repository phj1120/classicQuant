## 2. 백테스트 견고성

### 2.1 Walk-Forward / Out-of-Sample
- **현황**: `--walk-forward` 옵션 존재하나 IS vs OOS Sharpe 격차 표기 없음
- **보강**:
  - **Expanding window walk-forward**: 5년 학습 → 1년 시험 → 1년 롤
  - In-sample vs OOS Sharpe **격차 0.3 이내**여야 신뢰 가능
  - IS/OOS 비교 표 명시적 출력

### 2.2 파라미터 안정성 검증
임의 임계값들을 **민감도 표**로 만들어야 함:

| 파라미터 | 현재값 | 점검 범위 | 합격 기준 |
|---|---|---|---|
| `top_n` | 2 | 1~6 | Sharpe 변동 ≤ 20% |
| `portfolio_mdd_limit` | -0.18 | -0.10 ~ -0.30 | CAGR 단조 감소 |
| `mdd_filter_threshold` | -0.15 | -0.10 ~ -0.25 | 한 점에 몰리지 않음 |
| `mdd_threshold_ratio` | 0.8 | 0.5 ~ 1.0 | Sharpe ±0.1 이내 |
| `rolling_peak_window` | 252 | 126 ~ 504 | Sharpe ±0.1 이내 |
| `rebalance_threshold_pct` | 0.07 | 0.03 ~ 0.15 | 비용 후 우위 유지 |

→ 한 파라미터를 ±50% 흔들 때 Sharpe가 0.3 이상 변하면 **데이터스누핑**.

### 2.3 데이터 편향 제거
- **생존편향**: 현재 ETF 풀이 살아남은 종목들. 종료 ETF 포함 점검
- **선견편향**: alternative chain이 "당시 존재하던" 종목인지 검증 (EDV는 2007 출시)
- **시점 정렬**: 종가 기준 신호로 같은 날 매매하면 look-ahead. **T+1 체결** 가정 명시

### 2.4 통계적 유의성
- **Bootstrap**: 일별 수익을 1만 회 재추출 → CAGR/Sharpe **신뢰구간** 산출
- **Probabilistic Sharpe Ratio (PSR)**: 0.95 이상이어야 "Sharpe>0이 우연 아님"
- **Deflated Sharpe**: 15개 전략 시도 보정 후에도 유의해야 함

### 2.5 레짐별 분해
| 레짐 | 기간 예 | 점검 |
|---|---|---|
| 닷컴 붕괴 | 2000~2002 | 데이터 확장 필요 (현재 2006~) |
| GFC | 2008~2009 | 보유 |
| QE 강세장 | 2010~2019 | 보유 |
| 코로나 | 2020 | 보유 |
| 인플레·금리쇼크 | 2022 | 보유 |
| 정상화 | 2023~ | 보유 |

→ **2022년 채권·주식 동반 하락 국면**에서 각 전략 성과 별도 표기 필수

## 8. 합격 기준 (Go/No-Go 게이트)

펀드화 결정 전 충족 필수:

- [ ] 라이브 운용 **24개월 이상**
- [ ] 라이브 NAV vs 모델 NAV 괴리 **연 1.5%p 이내**
- [ ] OOS Sharpe ≥ **0.7**, IS-OOS 격차 < 0.3
- [ ] DSR ≥ **0.95**
- [ ] 모든 파라미터 ±30% 흔들어도 Sharpe 변동 < 20%
- [ ] 2022년형 채권·주식 동반 하락 시뮬레이션에서 MDD < **-20%**
- [ ] KIS API 장애 SOP 1회 이상 실전 점검
- [ ] 외부 회계 NAV verification 1회
