# classicQuant 투자 운용 가이드

> 작성일: 2026-03-22
> 전문 퀀트 투자 운용사 관점의 전략 분석 및 투자금별 운용 권고안

---

## 1. 프로젝트 총평 (전문가 관점)

### 강점

| 항목 | 평가 |
|------|------|
| 전략 다양성 | ★★★★★ — Keller(7개), Faber(2개), Antonacci(1개), 정적배분(3개), LAA(1개) 총 15개 |
| 방어 메커니즘 | ★★★★☆ — 카나리아 필터, MDD 서킷브레이커(-20%), 개별 전략 낙폭 필터(-5%) |
| 상관관계 관리 | ★★★★★ — `corr_constrained` 기준: Sharpe 상위 + 상관계수 0.7 미만 필터 |
| 거래비용 절감 | ★★★★☆ — 5% 리밸런싱 임계값, $5 최소 거래금액, 저비용 대체 ETF 체인 |
| 자동화 | ★★★★☆ — GitHub Actions 자동 실행, KIS API 연동 |

### 아키텍처상 주목할 점

1. **전략 선택 메타레이어**: 15개 전략 자체가 포트폴리오 → 전략 간 분산으로 "운이 나쁜 전략"의 영향을 구조적으로 제한
2. **대체 티커 체인**: SPY→SPLG, TLT→EDV→SPTL 등 주가 높은 ETF를 소액으로도 매수 가능하도록 설계
3. **NAV 기반 선택**: 실제 누적 수익률로 전략을 평가 → 백테스트 과적합 방지
4. **서킷브레이커**: 포트폴리오 MDD -20% 도달 시 Permanent Portfolio 단독 전환 → 재앙적 손실 방어

### 주의사항

- **워밍업 기간 필요**: 전략 선택 기준(`corr_constrained`)은 최소 1년(252거래일) NAV 데이터 필요
- **실제 수익률 괴리**: 학술 논문 백테스트 대비 매수 슬리피지, KIS 수수료 미반영 가능성
- **한국 세금**: 해외 ETF 양도소득세 22% (250만원 기본공제 후) 별도 고려 필요
- **환율 리스크**: 원/달러 환율 변동이 수익률에 직접 영향

## 3. 투자금별 운용 권고안

### 전제 조건

- 환율 가정: USD/KRW = 1,400원
- 대체 ETF 사용 (SPY→SPLG $72, TLT→SPTL $31, GLD→GLDM $51 등)
- KIS API를 통한 해외주식 매매
- 월 1회 리밸런싱 기준

---

### Tier 1: 소액 입문 (100만원 ~ 400만원 / ~$700 ~ $2,800)

#### 운용 현실

- 전략당 할당 가능 금액: $175~$700
- 대부분 ETF 1주 이상 매수 가능하나, 4개 전략 동시 운용 시 분산 효과 미미
- 거래비용 비중이 상대적으로 높음

#### 권고 설정

```json
{
  "selection": {
    "criteria": "corr_constrained",
    "top_n": 1,
    "mdd_filter_threshold": -0.10,
    "fallback_strategy": "all_weather",
    "portfolio_mdd_limit": -0.15
  },
  "strategy": {
    "rebalance_threshold_pct": 0.10,
    "cash_buffer_pct": 0.05,
    "min_trade_value_usd": 30.0
  }
}
```

#### 운용 방식

| 항목 | 내용 |
|------|------|
| 활성 전략 수 | 1개 (top_n=1) |
| 추천 전략 조합 | `strategies` 배열에서 **All Weather + Permanent + Golden Butterfly + VAA + GEM** 5개만 유지 |
| 리밸런싱 주기 | 월 1회 (하지만 5% → 10% 임계값으로 잦은 거래 방지) |
| 기대 수익률 | 연 8~12% (단일 전략 의존) |
| 기대 MDD | -10~-20% |

#### 유의사항

- 워밍업 1년 동안은 `offensive_mode` 기준으로 시작 권장
- 소액이므로 정적 전략(All Weather)으로 시작 후 데이터 축적 뒤 모멘텀 전략 전환
- **최소 투자금 현실**: SPY 1주 $570 → SPLG 사용 시 $72, GLDM $51 → 단일 정적 전략은 ~$300 이상이면 가능

---

### Tier 2: 시작 투자 (400만원 ~ 1,400만원 / $2,800 ~ $10,000)

#### 운용 현실

- 전략당 할당 가능 금액: $700~$2,500 (top_n=2 기준)
- 2개 전략 동시 운용으로 기초 분산 가능
- 대부분의 ETF 1주 이상 매수 가능

#### 권고 설정

```json
{
  "selection": {
    "criteria": "corr_constrained",
    "top_n": 2,
    "mdd_filter_threshold": -0.07,
    "fallback_strategy": "permanent",
    "portfolio_mdd_limit": -0.18
  },
  "strategy": {
    "rebalance_threshold_pct": 0.07,
    "cash_buffer_pct": 0.03,
    "min_trade_value_usd": 20.0
  }
}
```

#### 운용 방식

| 항목 | 내용 |
|------|------|
| 활성 전략 수 | 2개 (각 50%) |
| 추천 전략 풀 | 전체 15개 유지 (상관관계 필터가 자동으로 다양성 확보) |
| 리밸런싱 주기 | 월 1회 |
| 기대 수익률 | 연 9~14% |
| 기대 MDD | -12~-20% |

#### 전략 조합 예시 (corr_constrained 선택 시 자주 등장하는 조합)

```
VAA (공격적 모멘텀)  +  Permanent (안정적 방어)
DAA (분산 모멘텀)   +  All Weather (리스크 패리티)
GEM (단순 절대 모멘텀) + Golden Butterfly (분산 정적)
```

### Tier 3: 기본 투자 (1,400만원 ~ 4,200만원 / $10,000 ~ $30,000)

#### 운용 현실

- 전략당 할당: $2,500~$5,000 (top_n=3 기준)
- 3개 전략으로 실질적 전략 다각화 효과 시작
- 대부분 기능을 정상적으로 활용 가능

```json
    "top_n": 3,
    "mdd_filter_threshold": -0.05,
    "fallback_strategy": "permanent",
    "portfolio_mdd_limit": -0.20
  },
  "strategy": {
    "rebalance_threshold_pct": 0.05,
    "cash_buffer_pct": 0.02,

}
```

#### 운용 방식

| 항목 | 내용 |
|------|------|
| 활성 전략 수 | 3개 (각 33.3%) |
| 전략 풀 | 전체 15개 |
| 리밸런싱 주기 | 월 1회 |
| 기대 수익률 | 연 10~15% |
| 기대 MDD | -10~-18% |

#### 세금 최적화 포인트

- 연간 해외주식 양도차익 **250만원 비과세** 한도 적극 활용
- 손실 실현 (Tax-Loss Harvesting): 연말 평가손실 구간에서 매도 → 재매수 고려
- 기대 수익의 22% 세금 부담을 감안한 실질 수익률 계산 필요

- **리밸런싱 타이밍**: 월말 이후 첫 영업일 집중 실행 (배당락 영향 최소화)
- **연간 세금 계획**: 양도차익이 250만원 초과 예상 시 11~12월 중 세금 계획 수립

```json
    "mdd_filter_threshold": -0.05,
    "fallback_strategy": "all_weather",
    "portfolio_mdd_limit": -0.15

    "cash_buffer_pct": 0.05,
```

#### 추가 전략 고려사항

| 항목 | 권고 |
|------|------|

| 법인 계좌 | 투자금 5억 이상 시 법인 설립을 통한 세금 최적화 검토 |

## 5. 리스크 관리 프레임워크

### 5-1. 시스템 내 방어 레이어

```
레이어 1: 개별 전략 레벨
  └─ 카나리아/모멘텀 조건 → 수비자산(SHY, IEF, LQD)으로 자동 전환

레이어 2: 전략 선택 레벨
  └─ mdd_filter_threshold(-5%) → 낙폭 큰 전략 제외
  └─ corr_constrained → 상관관계 0.7 이상 전략 중복 제외

레이어 3: 포트폴리오 레벨
  └─ portfolio_mdd_limit(-20%) → 전체 낙폭 초과 시 Permanent 단독 전환

레이어 4: 거래 레벨
  └─ rebalance_threshold_pct(5%) → 노이즈 거래 방지
  └─ min_trade_value_usd($5) → 소액 주문 방지
```

### 5-2. 시스템 외 리스크 관리

| 리스크 | 대응 방안 |
|--------|-----------|
| 시스템 장애 | GitHub Actions 알림 설정, 수동 점검 루틴 |
| KIS API 장애 | 재시도 로직 존재(5회), 수동 리밸런싱 대기 |
| 환율 급변 | 원화 예비자금 10~20% 유지 |
| 블랙스완 이벤트 | 서킷브레이커 발동 후 Permanent 전환, 재진입 시점 수동 판단 |
| 세금 폭탄 | 연간 실현 이익 모니터링, 분기별 세금 예약 적립 |

### 5-3. 투자 기간별 기대 성과 시뮬레이션

```
초기 투자: 2,000만원 (Tier 2, top_n=2)
가정: 연 12% 수익, 연 2% 세후 조정

1년 후: 약 2,200만원
3년 후: 약 2,750만원
5년 후: 약 3,400만원
10년 후: 약 6,200만원

→ S&P 500 단순 투자(연 10%) 대비 전략 분산의 장점:
  - 하락장 방어 (Keller 전략의 수비 전환)
  - 유사 수익률에서 낮은 변동성
  - 심리적 안정으로 장기 보유 가능
```

## 7. FAQ

**Q. 백테스트 없이 바로 실거래 시작해도 되나요?**
A. `run_backfill.py`로 과거 데이터를 충분히 수집하면 시스템이 자동으로 NAV를 시뮬레이션합니다. 최소 1년치 데이터가 쌓이면 `corr_constrained` 전략 선택이 의미 있게 작동합니다.

**Q. 워밍업 기간(1년) 동안은 어떻게 하나요?**
A. `criteria`를 `"offensive_mode"`로 설정하거나, 전략 풀을 정적 배분(All Weather, Permanent)만으로 축소하여 운용하다가 데이터 축적 후 전환 권장.

**Q. top_n=4인데 선택된 전략이 4개 미만이면?**
A. 부족한 슬롯은 `fallback_strategy`(Permanent)로 보완됩니다. 25% 비중이 Permanent에 추가 배분됨.

**Q. GEM 전략이 자주 AGG에 투자하는 건 정상인가요?**
A. 정상입니다. GEM은 SPY vs EFA 중 상대 승자를 선택하되, 12개월 절대수익률 < 0이면 100% AGG 투자합니다. 약세장 방어 설계입니다.

**Q. 최적 선택 기준은 `corr_constrained`가 맞나요?**
A. 운용 경험상 `corr_constrained`(기본값)가 단순 Sharpe 대비 실제 상관관계를 낮추어 분산 효과를 극대화합니다. 특히 Keller 전략들 간 상관계수가 높아(0.7+), 이 필터 없이는 유사 전략 중복 선택 위험이 있습니다.

## 9. 핵심 원칙 요약

1. **워밍업부터**: 최소 1년 데이터 축적 후 `corr_constrained` 전략 선택 활성화
2. **투자금에 맞는 top_n**: 소액일수록 top_n을 줄여 전략당 배분 금액 확보
3. **리밸런싱 임계값 지키기**: 잦은 거래는 수수료 + 세금 부담 → 5% 이상 이탈 시에만 조정
4. **서킷브레이커 신뢰**: -20% 도달 시 시스템이 자동 방어 → 공황 매도 금지
5. **세금은 수익의 일부**: 해외 ETF 양도소득은 발생 시점이 아닌 실현 시점 과세 → 장기 운용이 유리
6. **환율은 리스크이자 수익**: 달러 자산 보유 자체가 환율 다각화 효과
